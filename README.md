# FastAPI基礎セットアップ課題

API開発における重要な概念（バリデーション、エラーハンドリング、高度なレスポンス制御）を学習するためのバックエンドAPIプログラムで、
現在は Swagger UI を通じてデータのやり取りが正しく行われることを確認する段階です。
「お店の自動受付システム」のような役割を持ち、ユーザーからのリクエストが正しいルールに従っているかをチェックし、適切な応答を返す仕組みをシミュレートしています。

---

## 実施内容

スクールのカリキュラムに沿って、以下を実装しました。

1. **基本エンドポイント**: GET リクエストによる疎通確認（Hello World）
2. **クエリパラメータ**: パス/クエリパラメータによる動的データ処理
3. **データ検証 (Pydantic)**: リクエストボディのバリデーションと POST 実装
4. **エラーハンドリング**: `HTTPException` によるカスタムエラー（400エラー）
5. **詳細なバリデーション**: `Query` による文字数（3〜50）や数値制限（1以上）
6. **レスポンスの多様化**: ステータスコード変更、HTML・テキスト応答の切り替え
7. **カスタムヘッダー**: `Response` オブジェクトによる追加ヘッダー付与

---

## 使用技術

* Python 3.x
* FastAPI
* Uvicorn (ASGI サーバー)
* Pydantic (データバリデーション)
* FastAPI Responses（HTML/Text対応）

## セットアップと起動方法

このプロジェクトをローカル環境で実行する手順です。

### 1. 仮想環境の作成

```bash
python -m venv venv
```

### 2. 仮想環境の有効化

**Windows**

```bash
venv\Scripts\activate
```

**Mac / Linux**

```bash
source venv/bin/activate
```

### 3. 必要パッケージのインストール

```bash
pip install -r requirements.txt
```

### 4. 起動する

```bash
uvicorn main:app --reload
```

### 5. 動作確認

ブラウザで以下にアクセス：

```
http://127.0.0.1:8000/docs
```

Swagger UI にて各エンドポイントの動作を確認できます。
