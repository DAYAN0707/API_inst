# FastAPI基礎セットアップ課題

API開発における重要な概念（バリデーション、エラーハンドリング、高度なレスポンス制御）を学習するためのバックエンドAPIプログラムで、
現在はSwagger UIを通じてデータのやり取りが正しく行われることを確認する段階です。<br>
「お店の自動受付システム」のような役割を持ち、ユーザーからのリクエストが正しいルールに従っているかをチェックし、適切な応答を返す仕組みをシミュレートしています。


## 実施内容

スクールのカリキュラムに沿って、以下の5つのステップを実装しました。

1. **基本エンドポイント**: GETリクエストによる疎通確認（ハローワールド）
2. **クエリパラメータ**: パスパラメータやクエリパラメータによる動的なデータ処理
3. **データ検証 (Pydantic)**: リクエストボディのバリデーションとPOSTリクエストの実装
4. **エラーハンドリング**: `HTTPException`によるカスタムエラーメッセージ（400エラー）の実装
5. **詳細なバリデーション**: `Query`を使用した文字数制限（3〜50文字）や数値の範囲指定（1以上）
6.  **レスポンスの多様化 (New!)**: 成功時のステータスコード変更（`201 Created`）、HTMLやプレーンテキストによるレスポンス形式の出し分け
1. **カスタムヘッダー (New!)**: `Response` オブジェクトを使用した、独自のHTTPヘッダー情報の付与

## 使用技術
* Python 3.x
* FastAPI
* Uvicorn (ASGI サーバー)
* Pydantic (データバリデーション)
* FastAPI Responses (HTML/Text対応)
